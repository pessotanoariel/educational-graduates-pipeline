# Publication Rules

## Purpose

These rules define what may and may not be included in the public version of the repository.

---

## Allowed Content

The following content may be published:

* Source code
* Documentation
* Project architecture
* Configuration templates
* Unit tests
* Project structure
* Roadmaps
* Technical backlog

---

## Restricted Content

The following content must never be published:

### Raw Data

Examples:

* Source datasets
* Operational exports
* Production extracts

---

### Processed Data

Examples:

* Processed datasets
* Consolidated datasets
* Curated datasets
* Published datasets

---

### Personal Information

Examples:

* Document numbers
* Names
* Email addresses
* Phone numbers
* Birth dates

---

### Sensitive Configuration

Examples:

* Credentials
* API keys
* Environment variables
* Private connection details

---

## Sample Data Policy

If sample datasets are required:

* Data must be synthetic
* Data must not originate from production systems
* Data must not contain personal information

---

## Repository Audit Requirement

Before any public release:

* Repository audit must be completed
* Privacy review must be completed
* Sensitive content review must be completed

---

## Publication Scope

The public repository is intended to demonstrate:

* Data engineering practices
* Data quality workflows
* Dataset consolidation logic
* Monitoring architecture
* Analytical modeling concepts

The repository is not intended to distribute educational graduate data.
